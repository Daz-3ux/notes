## 语义化版本号
- 主版本号（MAJOR）：当做了不兼容的 API 修改
- 次版本号（MINOR）：当做了向下兼容的功能性新增及修改。这里有个不成文的约定需要你注意，偶数为稳定版本，奇数为开发版本
- 修订号（PATCH）：当做了向下兼容的问题修正

## commit
```
<type>[optional scope]: <description>
// 空行
[optional body]
// 空行
[optional footer(s)]
```
- scope 必须用 () 括起来
- description 以动词开始
